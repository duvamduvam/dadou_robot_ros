# Software Overview

## Code structure
- `robot/nodes/`: ROS 2 boundary — subscribe, decode the JSON payload
  (`payload.py`, invalid payloads rejected with an explicit log), tick at 20 Hz
  (`TICK_PERIOD_S` in `robot_static.py`), delegate to an action.
- `robot/actions/`: one `Action` (ABC `action.py`: `update(msg)` /
  `process()`) per subsystem. Servos move through a linear ramp with I2C
  anti-spam and an animation deadman; the LED strip is driven through
  `FastNeoPixel` (vendored direct `_rpi_ws281x` write — the stock Blinka
  backend resolution is broken in the container; the ~31 ms frame-completion
  sleep per `show()` is KEPT, see incident 2026-07-13).
- `robot/sequences/`: `Track` (the single keyframe-track class, injected
  clock) and `AnimationManager` (samples a JSON sequence, one Track per body
  track).
- `robot/visual/`: `ImageMapping` (physical LED wiring as a precomputed table,
  locked by tests), `FastNeoPixel`, `Visual` (raw PNG loading).
- `robot/tests/unit/`: the 443-test suite (see below). `robot/tests/sandbox/`
  holds manual hardware benches only (not collected).

## Configuration & assets
- `robot/robot_config.py`: deploy constants (pins, channels, flags — including
  `WHEELS_CMD_VEL_ENABLED`). `robot/robot_static.py`: constants owned by the
  robot; shared constants live in `dadou_utils_ros/utils_static.py`
  (one-way migration rule, contract test `test_contrat_utils.py`).
- `json/`: sequences, expressions, lights — ALL under data-contract tests
  (structure, sorted keyframes, referenced assets must exist, PNG sizes).
- `medias/`: visuals (mouth 24x16, eye 8x8) and audio; the expression visuals
  are GENERATED by `conf/scripts/generate_expressions.py` (deterministic,
  includes the calibration test patterns `calib`, `calib-couleurs`,
  `calib-bords`).

## Extending
1. New expression: add it to `generate_expressions.py` (draw the PNG **as seen
   from the front** — the wiring table handles the physical layout), run the
   script, tests validate the references automatically.
2. New sequence: JSON under `json/sequences/<show>/`; keyframes are
   `[t_rel 0..1, value]`, validated by `test_sequences_json.py` /
   `test_sequences_assets.py`.
3. New actuator behaviour: extend the relevant `robot/actions/` module —
   respect the `Action` contract, keep hardware imports deferred
   (pattern documented in `wheels.py`) so the module stays importable on x86.
4. Always add unit tests; physical acquisitions (wiring, calibration) must be
   LOCKED by a test once verified on the robot.

## Tests & CI
- `.venv/bin/pytest -q` → 443 tests, no hardware needed (deferred imports +
  fakes). `pytest.ini` collects `robot/tests/unit` only.
- GitHub Actions runs the suite on each push; `dadou_utils_ros`' CI re-runs
  the consumer repos' tests against the current shared lib.

## Dependencies
- ROS 2 **Jazzy** (containerised; the app also runs against Gazebo Harmonic in
  simulation — see `conf/docker/sim/`).
- `dadou_utils_ros`: logging, `TimeUtils`, file utils, deployment (Ansible).
- `robot_interfaces` message package (`StringTime`).

## Logging
- Shared logging factory; application log at `/home/ros2_ws/log/robot.log`
  inside the container. Hot-loop logs are `debug` (SD card wear at 20 Hz).
